# 설정 탭 케이스

## 진입 조건
- 상단 탭에서 [⚙️ 설정] 클릭
- 로그인 완료 상태

## Case 1: 워크스페이스 관리
- 조건: 기본 설정 화면 (첫 번째 섹션)
- 표시:
  - 현재 워크스페이스 표시 + 전환 드롭다운
  - 워크스페이스 매핑 테이블:
    - 서브도메인명 | 환경 입력 필드 (자유 입력: dev, feature-login, qa 등)
    - 등록된 모든 서브도메인이 나열됨
    - 각 행에서 환경 직접 입력 가능
  - [+ 새 워크스페이스] 버튼
  - [워크스페이스 삭제] 버튼 (기본 워크스페이스는 삭제 불가)
- 인터랙션:
  - 환경 입력값 변경 → 즉시 저장
  - [+ 새 워크스페이스] → 이름 입력 모달 → 생성 (기본 전부 dev)
  - [삭제] → 확인 후 삭제

## Case 2: AI 설정
- 조건: 두 번째 섹션
- 표시:
  - **Reasoning 모델** 섹션 (읽기 전용):
    - Provider: 텍스트 표시 (예: OpenAI) — 읽기 전용, 회색 배경
    - Model: 텍스트 표시 (예: gpt-4o) — 읽기 전용, 회색 배경
  - **Fast 모델** 섹션 (읽기 전용):
    - Provider: 텍스트 표시 (예: OpenAI) — 읽기 전용, 회색 배경
    - Model: 텍스트 표시 (예: gpt-4o-mini) — 읽기 전용, 회색 배경
  - 안내 문구: "AI 모델 설정은 서버 배포 설정(yml/env)에서만 변경 가능합니다"
  - "다음 액션" 힌트: ON / OFF 토글
- 인터랙션:
  - AI 모델 섹션은 인터랙션 없음 (전체 읽기 전용 — 현재 적용된 설정 확인용)
  - 토글 변경 → 즉시 저장

## Case 3: Agent Loop 설정
- 조건: 세 번째 섹션
- 표시:
  - 최대 반복 횟수: 숫자 입력 (기본 20)
  - 턴당 Tool Call: 숫자 입력 (기본 5)
  - 타임아웃: 숫자 입력 + 초 (기본 120)
- 인터랙션:
  - 값 변경 → [저장] 버튼 클릭으로 반영

## Case 4: 계정 정보
- 조건: 네 번째 섹션
- 표시:
  - 이메일 (읽기 전용)
  - 이름 (읽기 전용)
  - 비밀번호 변경: [비밀번호 변경] 버튼
  - OTP 설정: 활성/비활성 토글
  - [로그아웃] 버튼
- 인터랙션:
  - [비밀번호 변경] → 현재 비밀번호 + 새 비밀번호 입력 폼
  - OTP 토글 ON → OTP setup API 호출 → otpauth:// URI 표시 + 6자리 코드 입력 → 인증 성공 시 활성화
  - OTP 토글 OFF → OTP 비활성화 API 호출
  - [로그아웃] → 세션 종료 + 로그인 페이지로 이동

## API 연동

| 동작 | API | 설명 |
|------|-----|------|
| 워크스페이스 목록 | GET /api/v1/workspaces | 유저의 워크스페이스 목록 |
| 워크스페이스 생성 | POST /api/v1/workspaces | 새 워크스페이스 |
| 워크스페이스 수정 | PUT /api/v1/workspaces/{id} | 매핑 변경 |
| 워크스페이스 삭제 | DELETE /api/v1/workspaces/{id} | 삭제 |
| 설정 조회 | GET /api/v1/settings | AI 티어별 설정 (전체 읽기 전용), Agent Loop 설정 |
| 설정 수정 | PUT /api/v1/settings | Agent Loop 설정 + 다음 액션 힌트 변경 (AI 모델 설정은 변경 불가) |
| 비밀번호 변경 | PUT /api/v1/auth/password | 비밀번호 변경 |
| OTP 설정 | POST /api/v1/auth/otp/setup | TOTP secret 생성 + QR URI 반환 |
| OTP 활성화 | POST /api/v1/auth/otp/verify | OTP 코드 검증 후 활성화 |
| OTP 비활성화 | DELETE /api/v1/auth/otp | OTP 비활성화 |
