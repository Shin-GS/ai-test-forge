# 채팅 화면 케이스

## 진입 조건
- 로그인 완료 후 메인 화면 (기본 탭)
- 좌측 사이드바에 세션 히스토리, 우측에 채팅 영역

## Case 1: 첫 접속 (빈 화면)
- 조건: 현재 활성 세션 없음 (새 대화 시작 전)
- 표시:
  - 중앙에 온보딩 영역
  - "🚀 빠른 시작" 타이틀
  - 퀵 액션 버튼들 (자주 쓰는 레시피 기반)
  - "또는 자유롭게 입력하세요..." 안내 문구
  - 하단 입력창 (활성)
- 인터랙션:
  - 퀵 액션 버튼 클릭 → 해당 레시피 실행 시작 (메시지 자동 입력 + 전송)
  - 입력창에 직접 타이핑 + 전송 → 새 세션 생성 + AI 응답 시작
- 좌측 사이드바:
  - 이전 세션 목록 (날짜 + 첫 메시지 요약)
  - "+ 새 대화" 버튼

## Case 2: 대화 진행 중 (Agent Loop 실행)
- 조건: 사용자가 메시지 전송 후 AI가 처리 중
- 표시:
  - 사용자 메시지 (우측 정렬, 보라색 버블)
  - AI 응답 영역 (좌측 정렬):
    - 텍스트 응답 (있으면)
    - Progress Steps 카드:
      - "🔄 API 호출 중... (N/M)"
      - 각 Step: ✅ 완료 / ⏳ 진행 중 / ⬜ 대기
      - Step에 서브도메인명 + HTTP method + path 표시
      - 완료된 Step에는 결과 요약 한 줄
  - 하단 입력창: **비활성** (AI 응답 대기 중 추가 입력 방지)
- 인터랙션:
  - 없음 (기다리기만)
  - 스크롤은 자동으로 하단 추적

## Case 3: 대화 완료 (결과 표시)
- 조건: Agent Loop 완료, AI가 최종 메시지 전달
- 표시:
  - AI 완료 메시지:
    - "✅ {작업명} 완료"
    - 결과 목록 (bullet point: 생성된 데이터 ID, 이름 등)
    - [상세 보기 ▾] 접기/펼치기 (각 API 호출의 요청/응답 상세)
  - (선택) "💡 더 할 수 있는 것" 버튼
    - 클릭 시 AI가 관련 다음 액션 제안
  - 하단 입력창: 활성 (다음 메시지 입력 가능)
- 인터랙션:
  - [상세 보기] 클릭 → Tool Call 상세 펼침 (요청 body, 응답 body)
  - [더 할 수 있는 것] 클릭 → AI 호출하여 추천 (버튼 형태로 표시)
  - 추천 버튼 클릭 → 해당 메시지 자동 전송
  - 입력창에 새 메시지 → 같은 세션에서 대화 이어감

## Case 4: 인증 필요 (401 에러)
- 조건: Agent Loop 중 서브도메인 API가 401 반환
- 표시:
  - Progress Steps에서 해당 Step이 ❌ 표시
  - Alert 카드 (에러 스타일):
    - "⚠️ {서브도메인명}에 로그인이 필요합니다."
    - [웹에서 로그인 →] 버튼 (login-page-url로 새 탭 열기)
    - "로그인 완료 후:" [계속 진행] 버튼
  - 하단 입력창: 비활성
- 인터랙션:
  - [웹에서 로그인 →] 클릭 → 새 탭에서 로그인 페이지 열림
  - [계속 진행] 클릭 → 중단된 Step부터 Agent Loop 재개
  - 인증 프로필이 여러 개면 어떤 프로필로 로그인할지 선택지 표시

## Case 5: 에러 발생 (서버 에러, 타임아웃)
- 조건: Agent Loop 중 5xx 에러, 네트워크 에러, 타임아웃
- 표시:
  - Progress Steps에서 해당 Step이 ❌ 표시
  - Alert 카드 (에러 스타일):
    - "❌ {서브도메인명} {method} {path} 실패"
    - 에러 원인 추정 + 해결 가이드
  - AI가 자동으로 재시도 판단 → 재시도 중이면 "🔄 재시도 중..." 표시
  - 3회 이상 실패 시: "반복 실패로 중단합니다. 다시 시도하시겠어요?" + [재시도] 버튼
  - 하단 입력창: 활성 (사용자가 다른 지시 가능)
- 인터랙션:
  - [재시도] 클릭 → 실패한 Step부터 재개
  - 입력창에 메시지 → AI에게 추가 지시 (예: "다른 방법으로 해봐")

## Case 6: 레시피 실행
- 조건: 레시피 탭에서 "실행" 클릭, 또는 채팅에서 "XX 레시피 실행해줘"
- 표시:
  - 시스템 메시지: "📋 레시피 '{레시피명}' 실행"
  - 변수 입력 필요 시: AI가 질문 (예: "포지션 ID를 입력해주세요")
  - 변수 없으면: 바로 실행 시작
  - Progress Steps (레시피 단계 기반):
    - 레시피 Step 1, 2, 3... 순서대로 표시
    - "📋" 아이콘으로 일반 Agent Loop과 구분
  - 완료 시: "✅ 레시피 실행 완료 (AI 비호출, 직접 실행)"
- 인터랙션:
  - 변수 입력 → 전송 → 실행 시작
  - 실패 시: Case 4, 5와 동일한 에러 처리

## 공통 요소

### 상단 헤더
- 로고 (🔨 AI Test Forge)
- 워크스페이스 드롭다운 (🌐 {워크스페이스명} ▾) — 워크스페이스 1개면 숨김
- 유저 아바타/메뉴

### 탭 네비게이션
- [💬 채팅] [📡 서브도메인] [📋 레시피] [⚙️ 설정]
- 채팅 탭일 때만 좌측 세션 사이드바 표시

### 좌측 사이드바 (세션 목록)
- 현재 활성 세션: ◉ 표시
- 이전 세션: 날짜 + 첫 메시지 앞 20자
- [+ 새 대화] 버튼

### 하단 입력창
- 텍스트 입력 영역 (자동 높이 조절, 최대 5줄)
- [전송] 버튼 (입력 있을 때만 활성)
- AI 처리 중: 입력 비활성 + "AI 응답 대기 중..." placeholder

## API 연동

| 동작 | API | 설명 |
|------|-----|------|
| 메시지 전송 | POST /api/v1/chat/{sessionId}/messages | SSE 연결 시작됨 |
| SSE 수신 | GET /api/v1/chat/{sessionId}/stream | tool_call_start, message, step_progress, done, error |
| Tool Call 결과 전달 | POST /api/v1/chat/{sessionId}/tool-result | FE가 서브도메인 호출 후 결과 전달 |
| 세션 목록 | GET /api/v1/chat/sessions | 좌측 사이드바용 |
| 새 세션 생성 | POST /api/v1/chat/sessions | 첫 메시지 전송 시 자동 |
