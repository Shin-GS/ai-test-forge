# 레시피 탭 케이스

## 진입 조건
- 상단 탭에서 [📋 레시피] 클릭
- 로그인 완료 상태

## Case 1: 레시피 목록 (정상)
- 조건: 저장된 레시피가 있음
- 표시:
  - 상단: 검색 바 + 태그 필터 드롭다운
  - "자주 사용" 섹션: 사용 횟수 순 상위 레시피
  - "전체" 섹션: 모든 레시피 카드
  - 각 레시피 카드:
    - 레시피명 + 사용 횟수 (⭐ ×N)
    - 설명 (한 줄 요약: "회원 생성 → 이력서 생성 → 포지션에 지원")
    - 태그 목록 (#채용 #회원)
    - 단계 수 + 변수 수 표시
    - [▶ 실행] [상세 보기] 버튼
- 인터랙션:
  - 검색: 레시피명, 설명, 태그로 필터
  - 태그 필터: 특정 태그만 표시
  - [▶ 실행] 클릭 → 채팅 탭으로 이동 + 레시피 실행 시작
  - [상세 보기] 클릭 → 레시피 상세 모달/페이지

## Case 2: 레시피 상세
- 조건: 목록에서 [상세 보기] 클릭
- 표시:
  - 레시피명 + 설명
  - 단계별 시각화:
    - Step 1: {서브도메인} {method} {path} — 파라미터 변수 표시
    - Step 2: ...
    - Step N: ...
  - 변수 목록: {{auto}} / {{input:라벨}} 구분
  - 태그
  - 생성일, 마지막 사용일, 총 사용 횟수
  - [▶ 실행] [수정] [삭제] 버튼
- 인터랙션:
  - [▶ 실행] → 채팅으로 이동 + 실행
  - [수정] → 편집 모드 (단계 추가/삭제, 변수 수정)
  - [삭제] → 확인 모달 후 삭제

## Case 3: 빈 상태 (레시피 없음)
- 조건: 저장된 레시피가 없음
- 표시:
  - 중앙 빈 상태:
    - "저장된 레시피가 없습니다"
    - "채팅에서 작업 후 '레시피로 저장해줘'라고 말해보세요."
    - [채팅으로 이동] 버튼
- 인터랙션:
  - [채팅으로 이동] → 채팅 탭 전환

## API 연동

| 동작 | API | 설명 |
|------|-----|------|
| 레시피 목록 | GET /api/v1/recipes | 전체 레시피 (태그, 사용횟수 포함) |
| 레시피 상세 | GET /api/v1/recipes/{id} | 단계별 상세 |
| 레시피 삭제 | DELETE /api/v1/recipes/{id} | 삭제 |
| 레시피 수정 | PUT /api/v1/recipes/{id} | 단계/변수 수정 |
| 레시피 복제 | POST /api/v1/recipes/{id}/clone | 다른 사용자의 레시피를 내 것으로 복제 |
| 레시피 실행 | POST /api/v1/recipes/{id}/execute | SSE 스트림 반환 (step별 진행) |
| 레시피 스펙 검증 | POST /api/v1/recipes/{id}/validate | 각 step이 현재 스펙과 호환되는지 검증 |
| 대화 이력 기반 생성 | POST /api/v1/recipes/generate-from-session | tool_call 히스토리에서 레시피 자동 생성 |
| 유사 레시피 제안 | GET /api/v1/recipes/suggest?query=&maxResults= | 사용자 요청과 유사한 레시피 검색 |
