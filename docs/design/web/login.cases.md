# 로그인 페이지 케이스

## 진입 조건
- 비로그인 상태에서 아무 페이지 접근 시 자동 리다이렉트
- 또는 직접 `/login` 접근

## Case 1: 기본 로그인 폼
- 조건: 비로그인 상태
- 표시:
  - 중앙 정렬 카드 형태
  - 로고 + 서비스명
  - 이메일 입력 필드
  - 비밀번호 입력 필드
  - [로그인] 버튼
- 인터랙션:
  - 이메일 + 비밀번호 입력 → [로그인] 클릭 → 인증 요청
  - 성공 → 메인 화면(채팅)으로 이동
  - 실패 → 에러 메시지 표시

## Case 2: 로그인 실패
- 조건: 잘못된 이메일 또는 비밀번호 입력
- 표시:
  - 폼 상단에 에러 메시지: "이메일 또는 비밀번호가 올바르지 않습니다."
  - 입력 필드는 유지 (비밀번호만 초기화)
- 인터랙션:
  - 재입력 후 다시 시도

## Case 3: OTP 입력 (2단계 인증, 선택)
- 조건: 이메일/비밀번호 인증 성공 후, OTP가 설정된 계정
- 표시:
  - OTP 코드 입력 필드 (6자리)
  - [인증] 버튼
  - "인증 앱에서 코드를 확인하세요" 안내
- 인터랙션:
  - OTP 입력 → 인증 성공 → 메인 화면 이동
  - OTP 실패 → "인증 코드가 올바르지 않습니다." 에러

## API 연동

| 동작 | API | 설명 |
|------|-----|------|
| 로그인 | POST /api/v1/auth/login | email + password → JWT 반환 |
| OTP 검증 | POST /api/v1/auth/otp/verify | OTP 코드 → 최종 JWT 반환 |
