# 서브도메인 탭 케이스

## 진입 조건
- 상단 탭에서 [📡 서브도메인] 클릭
- 로그인 완료 상태

## Case 1: 서브도메인 목록 (정상)
- 조건: 등록된 서브도메인이 여러 개 있고, 대부분 ACTIVE
- 표시:
  - 상단: 검색 바 + 환경 필터 드롭다운 + 상태 필터 드롭다운
  - 서브도메인 카드 목록 (접힌 상태 기본):
    - 서비스명 + 상태 배지 (✅ ACTIVE / ⚠️ STALE) + 환경 수 + API 수
  - 카드 클릭 → 펼침 (환경별 상세 + 인증 상태 + 설명)
- 인터랙션:
  - 검색: 서비스명으로 필터링
  - 환경 필터: 전체 / dev / feature-* / qa
  - 상태 필터: 전체 / ACTIVE / STALE
  - 카드 클릭: 펼침/접힘 토글
  - 펼친 상태에서 환경 클릭: 상세 페이지로 이동 (API 목록)

## Case 2: 서브도메인 상세 (API 목록)
- 조건: 목록에서 특정 환경을 클릭
- 표시:
  - ← 뒤로 버튼 + 서비스명 (환경)
  - 상태, API 수, 마지막 갱신 시각, base-url
  - 서비스 설명 (OpenAPI info.description 또는 AI 자동 요약)
  - API 검색 바
  - API 목록 (태그별 그룹핑):
    - 📁 태그명 (N개)
    - 각 API: HTTP method + path + summary
  - 하단: [채팅에서 이 서버 사용하기 →] 버튼
- 인터랙션:
  - API 검색: path 또는 summary로 필터
  - 태그 접기/펼치기
  - [채팅에서 이 서버 사용하기] → 채팅 탭으로 이동 + "user-service의 POST /api/members 호출해줘" 자동 입력
  - ← 뒤로 → 목록으로 복귀

## Case 3: 빈 상태 (등록된 서브도메인 없음)
- 조건: 아무 서브도메인도 등록되지 않음
- 표시:
  - 중앙에 빈 상태 일러스트 + 메시지
  - "등록된 서브도메인이 없습니다"
  - "서브도메인 서버에 클라이언트 라이브러리를 추가하면 자동으로 등록됩니다."
  - [연동 가이드 보기] 버튼
- 인터랙션:
  - [연동 가이드 보기] → 별도 가이드 페이지 또는 모달

## Case 4: STALE 서버 있음
- 조건: 일부 서버가 STALE 상태 (heartbeat 미응답)
- 표시:
  - 목록 상단에 경고 배너: "⚠️ N개 서버의 연결이 불안정합니다"
  - 해당 서버 카드에 ⚠️ STALE 배지 + "10분 전 마지막 heartbeat"
  - STALE 서버는 목록 하단으로 정렬 (ACTIVE 우선)
- 인터랙션:
  - 경고 배너 닫기 가능
  - STALE 서버 카드에 "이 서버가 다시 시작되면 자동으로 복구됩니다" 안내

## API 연동

| 동작 | API | 설명 |
|------|-----|------|
| 서브도메인 목록 | GET /api/v1/specs | 등록된 전체 서브도메인 + 환경별 상태 |
| 서브도메인 상세 | GET /api/v1/specs/{name}/detail?environment={env} | 특정 서브도메인의 API 목록 (파싱된 엔드포인트) |
