# QA Test Checklist

수동 테스트 체크리스트. 기능 구현 후 배포 전 점검용.

## 사용법

1. `docs/test/index.html`을 브라우저에서 열기
2. Feature 드롭다운에서 기능 선택
3. 각 케이스의 Pass / Fail / Skip 버튼 클릭
4. 상태는 localStorage에 자동 저장

## 테스트 현황

| 기능 | 파일 | 우선순위 | 케이스 수 | 마지막 업데이트 |

## 우선순위 가이드

- **critical**: 배포 전 반드시 전체 통과 (핵심 플로우 — Agent Loop, FE 직접 호출, 레시피 실행)
- **high**: 관련 기능 변경 시 필수 확인 (서브도메인 등록/heartbeat, 인증, 레시피 CRUD)
- **medium**: 주기적 확인 (워크스페이스 전환, AI 설정 변경)
- **low**: 대규모 변경 시에만 (UI 디테일)

## 테스트 범위 정의

### critical — Agent Loop + FE 직접 호출

| 테스트 시나리오 | 전제 조건 | 기대 결과 |
|---------------|----------|----------|
| 단일 API 호출 | 서브도메인 1개 등록 + 로그인됨 | 채팅에서 요청 → FE가 API 호출 → 결과 표시 |
| 크로스 서비스 호출 (3개 이상) | 서브도메인 3개 등록 | 의존성 순서대로 호출 → 전체 성공 |
| 401 에러 → 로그인 유도 | 서브도메인 미로그인 | 로그인 페이지 링크 표시 + 계속 진행 동작 |
| 5xx 에러 → 재시도 | 서브도메인 서버 일시 장애 | 에러 표시 + 재시도 버튼 동작 |
| Agent Loop 타임아웃 | 타임아웃 120초 초과 상황 | 타임아웃 에러 표시 + 입력 활성화 |
| 동일 API 3회 실패 → 중단 | 반복 실패 상황 | "반복 실패로 중단" 메시지 + 재시도 제안 |
| SSE 연결 끊김 → 재연결 | 네트워크 일시 끊김 | 재연결 후 누락 이벤트 수신 |
| CORS 동작 확인 | 서브도메인 CORS 설정 | FE → 서브도메인 직접 호출 성공, 쿠키 전달 |

### high — 서브도메인 등록

| 테스트 시나리오 | 전제 조건 | 기대 결과 |
|---------------|----------|----------|
| 클라이언트 라이브러리 push 등록 | Spring Boot 서버 + 라이브러리 | 서버 시작 → 메인 서버에 자동 등록, ACTIVE |
| heartbeat 유지 | 등록 후 30초 대기 | heartbeat 수신, ACTIVE 유지 |
| heartbeat 중단 → STALE | 서브도메인 서버 종료 | 5분 후 STALE 표시, UI 경고 배너 |
| STALE → 자동 삭제 | 30분 대기 | 자동 삭제, 목록에서 제거 |
| 수동 업로드 등록 | OpenAPI JSON 파일 | 업로드 → 등록 → ACTIVE |
| 비동기 등록 (대형 스펙) | 5MB+ JSON | 202 Accepted → REGISTERING → ACTIVE |
| environment 분리 | 같은 서비스 2개 환경 | name+env별 별도 등록 |

### high — 레시피

| 테스트 시나리오 | 전제 조건 | 기대 결과 |
|---------------|----------|----------|
| 레시피 실행 (변수 없음) | 레시피 저장됨, 전체 auto | 즉시 실행 → 결과 표시 (AI 비호출) |
| 레시피 실행 (변수 있음) | {{input:포지션ID}} 변수 | 변수 질문 → 입력 → 실행 |
| 레시피 저장 (대화에서) | 대화로 작업 완료 후 | "레시피로 저장해줘" → 저장 완료 |
| 레시피 실행 중 401 | 서브도메인 미로그인 | 로그인 유도 + 계속 진행 |
| 레시피 유효성 검증 | 참조 API가 스펙에서 삭제됨 | 실행 전 경고 표시 |

### high — 로그인/인증

| 테스트 시나리오 | 전제 조건 | 기대 결과 |
|---------------|----------|----------|
| 이메일/비밀번호 로그인 | 계정 생성됨 | 로그인 성공 → 채팅 화면 |
| 잘못된 비밀번호 | 없음 | 에러 메시지 표시 |
| OTP 인증 성공 | OTP 활성화 계정 | 2단계 인증 → 로그인 |
| OTP 인증 실패 | 잘못된 코드 | 에러 메시지 |
| JWT 만료 → 재로그인 | 세션 만료 | 로그인 페이지로 리다이렉트 |

### medium — 워크스페이스

| 테스트 시나리오 | 전제 조건 | 기대 결과 |
|---------------|----------|----------|
| 워크스페이스 생성 | 없음 | 이름 입력 → 생성 (전부 dev 기본) |
| 환경 매핑 변경 | 워크스페이스 존재 | 드롭다운 변경 → 즉시 저장 |
| 워크스페이스 전환 | 2개 이상 | 상단 드롭다운에서 전환 → 서브도메인 환경 변경 반영 |
| 워크스페이스 삭제 | 기본이 아닌 워크스페이스 | 삭제 → 기본 워크스페이스로 전환 |
