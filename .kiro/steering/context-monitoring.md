# 컨텍스트 효율 규칙

## 목적
main 에이전트가 컨텍스트를 효율적으로 사용하도록 제어한다.

## 파일 읽기 규칙

- 변경 대상 파일과 직접 관련된 파일은 자유롭게 읽는다 (개수 제한 없음).
- **목적 없는 탐색적 읽기**(구조 파악, "일단 읽어보기")가 10개 이상 필요하면 `context-gatherer` 서브에이전트에 위임한다.
- 대형 파일(500줄+)은 필요한 라인 범위만 지정한다.
- 같은 파일을 2회 이상 읽지 않는다 — 첫 읽기에서 필요한 정보를 모두 추출한다.

## 서브에이전트 프롬프트 규칙

- 변경 대상 파일 경로와 핵심 컨텍스트만 포함
- 전체 코드를 프롬프트에 복사하지 않는다 (서브에이전트가 직접 읽도록 지시)
- 불필요한 배경 설명 최소화
- contextFiles로 경로만 전달, 인라인 코드 블록 금지
- 프롬프트 길이: 500자 이내 권장, 1000자 초과 금지

## 서브에이전트 결과 처리

- raw output 전체를 그대로 전달하지 않는다
- 핵심 결론/피드백만 구조화하여 전달
- 중복되는 내용은 제거
- 수용할 내용: "변경 파일 목록 + 핵심 요약 (3줄 이내)"
- 리뷰 피드백: 항목별 1줄 요약만 유지

## 연쇄 호출 규칙

- 서브에이전트 연쇄 호출은 **3단계까지** 허용
- 3단계 초과 시: 중간 결과를 요약하고, 사용자에게 진행 상황 보고 후 다음 단계 진행
- 병렬 호출 가능한 에이전트는 반드시 병렬로 호출 (순차 금지)
- 같은 에이전트를 같은 작업으로 2회 이상 호출하지 않는다 (무한 루프 방지)

## Tool 결과 처리

- JSON, HTML, 로그 등 raw payload를 그대로 사용자에게 전달하지 않는다
- 필요한 필드/섹션만 추출하여 전달
- 우선순위: 필요한 필드 extract → top-n → summarize
- grep/search 결과: 매칭 파일 경로 + 라인 번호만 유지, 주변 컨텍스트는 필요 시에만

## 대화 컨텍스트 규칙

- 이전 턴에서 이미 전달한 내용을 다시 반복하지 않는다
- 압축 시 필수 정보(ID, 날짜, 상태값, 에러 코드)는 보존한다
- 사용자에게 보고할 때: 변경 파일 목록 + 핵심 변경 내용 (파일당 1줄)
- 코드 블록은 핵심 변경 부분만 발췌 (전체 파일 출력 금지)

## 대규모 작업 판단 기준

아래 조건 중 하나라도 해당하면 "대규모 작업"으로 판단하고 이 규칙을 엄격히 적용:
- 서브에이전트 호출 3회 이상 예상
- 변경 대상 파일 10개 이상
- FE + BE 동시 변경
- 새 화면/기능 추가 (Phase 1~5 전체 워크플로우)
